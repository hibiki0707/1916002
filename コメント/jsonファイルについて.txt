jsonファイルについて・・・
cocos2d-xとかなら、jsonを読み取る
機能があったりするが、通常はない。
自前で用意するのは大変。
①jsonファイル読み取りライブラリを導入する。
②あらかじめC＃などで、バイナリに変換
するツールを作っておく

i.C++は文字列の機能がほかの言語に比べると貧弱
ii.文字解釈（字句解析）が思いのほか重い。

できるだけC++としてはバイナリで読みたい。
例えばテクスチャパッキングだったら、
切り抜き矩形情報と中心情報があればいいので、
それをstructにしてバイナリ化しておく
struct ClipPict{
      int x,y,x,h; // 16バイト
      bool totate; // 4バイト
      float cx,cy; // 8バイト
｝ // 28バイト


C＋＋はバイナリ読み込みの方が速い。↑の例なら
28バイトを一気にfreadで読み込んで書き込み先を
ClipPict型の変数に置けば一瞬で使える状態になる。

これに対して、テキストの場合は、いちいち字句解析が
必要になる。


Web系の開発では、バイナリが使えないので、ほうぼうで
jsonが使用されている。vrm/gltfという3Dデータ形式が
あるが、これはjsonをもとにしている。

vrm/gltfは非常にややこしい。前半がjsonで公判がバイナリ

jsonはWebとかでよく使われるデータ形式でJavascriot用
な感じ。でも昔はXMLというのが使われていた。
＜XML＞みたいので、HTML形式に近いのでブラウザと親和性
が高いということでブームになった。
Ajax←Javaｓcriptを用いてＸＭＬをやりとりする、
ＸＭＬは汎用でーだ形式なので、Ｗｅｂで通信に使いやすい。
ただし、送る文字列データが無駄にでかくなるので、
もう少し小さいのを使いたい→json


4バイトデータ
01　23　45　67

intとかだと、リトル（ビッグ？）エンディアンの場合
45　67　01　23

64　45　23　01

てな感じで、バイナリへの保存、読み出しのルールが
変わってしまう。なので、この手のデータのやり取りは
Webには向いていない。
